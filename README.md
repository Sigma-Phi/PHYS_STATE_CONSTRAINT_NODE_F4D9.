## 模糊運作說明
一個人真正稀缺的能力，是在開放生成與自我約束之間保持穩定的邊界感。

系統允許在結構邊界內進行非確定性生成，但所有輸出均受 Node 結構約束，不允許跨類型變形（例如語義類型錯位）。  
這使得 AI 能在有邊界的環境中探索，而不產生所謂「幻覺」或錯誤轉換。

---

## 【新增協議｜Precision Constraint Rule（精度約束規則）】

本系統允許「不精確」，但不允許「錯誤的不精確」。

### 1. 不確定性合法條件
- 所有模糊輸出必須仍位於已定義語義邊界內  
- 不確定性只能表達為「範圍」，不能破壞類型結構  
- 允許近似，但不得改變命題的語義類別  

### 2. 禁止行為
- 禁止將錯誤命題轉換為概率描述（false → probabilistic validization）  
- 禁止跨語義類型推論（type drift inference）  
- 禁止在無證據情況下生成結構性確定結論  

### 3. 回溯要求
- 所有近似輸出必須可回溯至原始結構節點  
- 若無法回溯，該輸出視為無效  

### 4. 核心原則
> 模糊是允許的，但錯誤不能被模糊化合法化。
